# AnimatorVariable Event System (动画机变量控制事件系统)

## 概述
动画机变量控制事件系统允许您在技能执行过程中动态地控制动画机（Animator）的变量，包括布尔值、整数、浮点数和触发器。这个系统可以用于实现复杂的动画状态切换、连击系统、条件分支等功能。

**🆕 新功能：智能动画机检测 + 便捷变量添加！系统会自动检测CombatController的动画机，并提供动画机变量的下拉选择功能。**

## 🚀 快速开始

1. **创建事件**: `Create > AbilityEvents > AnimatorVariable`
2. **选择角色**: 在场景中选中包含CombatController的角色对象 (可选)
3. **刷新变量**: 点击`🔄 刷新可用变量列表`按钮 (系统自动检测动画机)
4. **添加变量**: 从下拉列表选择要控制的变量，点击`添加选中的变量`
5. **设置数值**: 配置每个变量的目标值
6. **完成**: 事件会控制所有添加的变量

**💡 提示**: 
- 智能检测CombatController的动画机，无需手动指定！
- 从下拉列表选择变量，避免输入错误！
- 支持批量添加和自定义变量！

## 功能特性

### 支持的变量类型
- **Bool（布尔值）**: 用于开关状态控制
- **Int（整数）**: 用于数值状态控制
- **Float（浮点数）**: 用于精确数值控制
- **Trigger（触发器）**: 用于瞬时状态触发

### 事件类型
- **瞬时事件（EventTime）**: 在指定时间点设置变量值
- **持续事件（EventRange）**: 在指定时间段内保持变量值，并可选择在结束时恢复原值

### ✨ 核心功能
- **🎯 智能动画机检测**: 自动检测最合适的动画机，优先使用CombatController
- **📋 下拉选择变量**: 从动画机变量列表中选择，避免输入错误
- **➕ 灵活添加方式**: 支持单个添加、批量添加、自定义变量
- **🗑️ 便捷管理**: 可编辑变量名和类型，一键删除不需要的变量
- **🔄 原值恢复**: 可以选择在事件结束时恢复变量的原始值
- **🛡️ 安全检查**: 自动检查动画机中是否存在指定的变量
- **📡 事件回调**: 提供丰富的事件回调接口用于扩展功能
- **🖥️ 友好界面**: 自定义Editor界面，实时显示检测状态和统计信息

## 使用方法

### 1. 创建动画机变量事件
1. 在Project窗口中右键点击
2. 选择 `Create > AbilityEvents > AnimatorVariable`
3. 为事件命名（例如：`SetAttackState`）

### 2. 🆕 智能变量添加（推荐方式）

#### 快速开始
1. **🎯 智能动画机检测**（全自动）
   - 系统会按优先级自动检测动画机：
     1. 手动指定的目标动画机
     2. 当前选中对象的CombatController动画机  
     3. 场景中CombatController的动画机
     4. 场景中任意可用的动画机
   - **推荐**：选中包含CombatController的角色对象，系统自动检测

2. **刷新可用变量**
   - 点击 `🔄 刷新可用变量列表` 按钮
   - 系统会自动扫描动画机，获取所有可用的参数名称
   - 下拉列表将显示所有检测到的变量

3. **添加要控制的变量**
   - **方式1 - 从列表选择**: 在下拉列表中选择变量，点击`添加选中的变量`
   - **方式2 - 批量添加**: 点击`添加所有变量`一次性添加所有动画机变量
   - **方式3 - 自定义添加**: 手动输入变量名，点击`添加自定义变量`

4. **配置变量值**
   - 对于添加的每个变量：
     - **变量名**: 可以修改变量名
     - **变量类型**: 可以调整变量类型
     - **Bool**: 设置true/false值
     - **Int**: 设置整数值
     - **Float**: 设置浮点数值
     - **Trigger**: 选择激活或重置触发器
   - **恢复原值**: 选择是否在事件结束时恢复到原始值
   - **删除变量**: 点击 `×` 按钮删除不需要的变量

#### 智能功能
- **🚫 重复检测**: 防止添加重复的变量
- **🎯 类型自动设置**: 从动画机选择时自动设置正确的变量类型
- **📊 实时统计**: Inspector显示已添加变量数和可用变量数
- **⚠️ 智能提示**: 提示用户添加变量或解决配置问题

### 3. 事件类型配置

#### 基本设置
- **持续性事件**: 
  - `false` = 瞬时事件（在指定时间点触发）
  - `true` = 持续事件（在时间段内生效）
- **运行时自动刷新**: 是否在事件开始时自动刷新变量列表

### 4. 实际应用示例

#### 📋 使用流程示例
假设您有一个角色的动画机包含以下变量：
- `IsAttacking` (Bool)
- `ComboCount` (Int) 
- `AttackSpeed` (Float)
- `SpecialAttack` (Trigger)

**步骤：**
1. 创建AnimatorVariable事件
2. 选中角色对象（包含CombatController）
3. 点击"🔄 刷新可用变量列表"→ 系统自动检测到4个变量
4. 添加要控制的变量：
   - 从下拉列表选择`IsAttacking`，点击"添加选中的变量"
   - 从下拉列表选择`ComboCount`，点击"添加选中的变量"
5. 设置值：
   - `IsAttacking` = true，恢复原值 = true
   - `ComboCount` = 2，恢复原值 = false
6. 设置为持续事件
7. 完成！

#### 示例场景

**场景1：攻击状态控制**
- 添加变量：`IsAttacking` (Bool)
- 设置值：true
- 恢复原值：true
- 事件类型：持续事件
- 效果：在攻击期间设置为true，结束后恢复false

**场景2：连击计数更新**
- 添加变量：`ComboCount` (Int)
- 设置值：2
- 恢复原值：false
- 事件类型：瞬时事件
- 效果：将连击数设置为2并保持

**场景3：技能触发**
- 添加变量：`SpecialAttack` (Trigger)
- 激活触发器：true
- 事件类型：瞬时事件
- 效果：触发特殊攻击动画

**场景4：多变量同时控制**
- 批量添加：点击"添加所有变量"
- 删除不需要的变量，只保留需要控制的
- 分别设置每个变量的值
- 效果：一个事件同时控制多个动画机变量

## 事件回调接口

系统提供以下事件回调，可以在 `CombatEvent` 或其他实现了 `ICombatEventListener` 的类中监听：

```csharp
// 通用变量变更回调
void OnAnimatorVariableChanged(CombatController character, string variableName, object oldValue, object newValue);

// 布尔变量设置回调
void OnAnimatorBoolSet(CombatController character, string variableName, bool value);

// 整数变量设置回调
void OnAnimatorIntSet(CombatController character, string variableName, int value);

// 浮点变量设置回调
void OnAnimatorFloatSet(CombatController character, string variableName, float value);

// 触发器激活回调
void OnAnimatorTriggerSet(CombatController character, string triggerName);
```

## 最佳实践

### 1. 命名规范
- 使用描述性的变量名（如：`IsBlocking`, `ComboPhase`, `AttackSpeed`）
- 保持与动画机中的命名一致

### 2. 事件类型选择
- **瞬时事件**: 适用于状态切换、计数器更新等
- **持续事件**: 适用于需要在一段时间内保持状态的场景

### 3. 原值恢复
- 对于临时状态（如攻击状态），建议启用原值恢复
- 对于永久性变化（如经验值、等级），不要启用原值恢复

### 4. 性能考虑
- 避免在单个事件中设置过多变量
- 对于高频率触发的事件，考虑使用更轻量级的实现

## 调试和故障排除

### 常见问题
1. **变量不存在警告**: 检查动画机中是否确实存在该变量名
2. **类型不匹配**: 确保设置的变量类型与动画机中的类型一致
3. **事件不触发**: 检查事件时间设置和CombatController的配置

### 调试工具
- 系统会在Console中输出详细的调试信息
- 可以通过事件回调监听变量变更情况
- 在预览模式下可以看到将要设置的变量信息
