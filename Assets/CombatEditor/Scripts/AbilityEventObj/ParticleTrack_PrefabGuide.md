# 粒子轨道预制体创建指南

## 🎨 概述

为了与粒子轨道事件系统完美配合，需要正确设计和配置粒子预制体。本指南将详细介绍如何创建适用于轨道系统的粒子预制体。

## 🏗️ 基础预制体结构

### 推荐层级结构
```
ParticleEffectPrefab (根对象)
├── MainParticle (主粒子系统)
├── SecondaryParticle (次要粒子系统)
├── ChildEffects (子效果组)
│   ├── Sparks (火花效果)
│   └── Glow (光晕效果)
└── Audio (音效组件 - 可选)
```

### 根对象配置
- **名称**: 使用描述性名称，如 `SwordSlash_Effect`
- **Transform**: 位置设为 (0,0,0)，旋转和缩放为默认值
- **组件**: 只添加必要的组件，避免不必要的脚本

## ⚙️ 粒子系统配置指南

### 主要设置 (Main Module)

#### ✅ 推荐设置
```csharp
Duration: 2.0-5.0秒 (根据效果需要)
Looping: false (轨道系统会控制循环)
Start Lifetime: 合理的粒子生命周期
Start Speed: 适中的初始速度
Start Size: 根据视觉需求调整
Start Color: 使用有意义的默认颜色
Max Particles: 控制在合理范围内 (100-1000)
Simulation Speed: 1.0 (轨道系统会控制速度)
```

#### ⚠️ 注意事项
- **不要使用Play On Awake**: 轨道系统会控制播放时机
- **避免过短的Duration**: 可能导致轨道播放不完整
- **合理控制Max Particles**: 过多会影响性能

### 发射模块 (Emission)

#### 轨道兼容配置
```csharp
Rate over Time: 设置基础发射率
Rate over Distance: 可选，用于移动效果
Bursts: 可以使用，创建爆发效果
```

#### 强度控制支持
- 轨道系统会动态调整 `Rate over Time`
- 如果启用强度曲线，原始发射率会被用作基准值
- Bursts不受强度曲线影响

### 形状模块 (Shape)

#### 推荐形状类型
```csharp
Box: 适合区域效果 (如爆炸、光环)
Cone: 适合方向性效果 (如火焰、冲击波)
Circle: 适合环形效果 (如法阵、护盾)
Mesh: 适合复杂形状 (如武器轮廓)
```

### 速度模块 (Velocity over Lifetime)

#### 配置建议
- 使用合理的速度曲线
- 避免过于剧烈的速度变化
- 考虑父对象移动的影响

### 颜色模块 (Color over Lifetime)

#### 轨道优化
```csharp
// 推荐的颜色渐变模式
开始: 全透明或半透明
中间: 完全不透明，主要颜色
结束: 逐渐透明，可能改变色调
```

### 大小模块 (Size over Lifetime)

#### 常用模式
- **渐强**: 从小到大 (0.3 → 1.0)
- **渐弱**: 从大到小 (1.0 → 0.0)
- **脉冲**: 先大后小再大
- **稳定**: 保持恒定大小

## 🎯 特定效果类型的配置

### 1. 武器特效

#### 剑气/刀光
```
主粒子系统:
- Shape: Box 或 Cone
- Start Speed: 中等
- Color: 锐利的白色或蓝色
- Size: 渐变消失

次要效果:
- 火花粒子 (小型，快速)
- 光晕效果 (大型，缓慢消失)
```

#### 魔法武器
```
主效果:
- 神秘的颜色 (紫色、金色)
- 复杂的形状 (Mesh 或 多个Circle)
- 特殊材质 (发光、扭曲)

环境效果:
- 周围的能量波动
- 漂浮的符文或光点
```

### 2. 技能特效

#### 火焰类
```
主火焰:
- Shape: Cone
- Color: 红橙黄渐变
- Velocity: 向上扩散
- Noise: 添加随机扰动

烟雾:
- Shape: Cone (更大角度)
- Color: 深灰到透明
- Velocity: 向上漂浮
- Drag: 增加阻力
```

#### 冰霜类
```
主效果:
- Shape: Circle 或 Box
- Color: 蓝白渐变
- Size: 快速扩张然后稳定
- Material: 使用冰晶材质

环境效果:
- 冰晶粒子 (尖锐形状)
- 寒气效果 (缓慢漂浮)
```

#### 雷电类
```
主雷电:
- Shape: Box (线性)
- Color: 亮蓝白色
- Velocity: 不规则运动
- 闪烁效果

次要效果:
- 电弧粒子
- 能量残留
- 冲击波
```

### 3. 环境特效

#### 光环/法阵
```
主光环:
- Shape: Circle
- Color: 神圣的金色或白色
- Size: 稳定或脉冲
- Rotation: 缓慢旋转

符文效果:
- 浮现的符文纹理
- 能量流动效果
- 光线投射
```

## 📋 材质和纹理建议

### 材质选择
```csharp
Additive: 适合发光效果，能量类
Alpha Blend: 适合烟雾，普通粒子
Multiply: 适合阴影，暗色效果
```

### 纹理指南
- **分辨率**: 64x64 到 512x512
- **格式**: 使用压缩格式节省内存
- **Alpha通道**: 正确设置透明度
- **平铺**: 考虑纹理重复使用

## 🔧 性能优化建议

### 粒子数量控制
```
简单效果: 50-200 个粒子
中等效果: 200-500 个粒子
复杂效果: 500-1000 个粒子
特殊效果: 1000+ 个粒子 (谨慎使用)
```

### LOD (Level of Detail) 设计
```csharp
// 为不同距离创建不同版本
近距离: 完整效果，高质量
中距离: 简化版本，减少粒子
远距离: 最简版本，或完全隐藏
```

### 批处理优化
- 使用相同的材质和纹理
- 避免不必要的模块
- 合理设置Update模式

## 🎨 创建流程示例

### 步骤1: 创建基础结构
1. 创建空GameObject作为根对象
2. 添加ParticleSystem组件
3. 配置Main模块基础参数

### 步骤2: 设计主要效果
1. 配置Emission模块
2. 设置Shape模块
3. 调整Color和Size模块

### 步骤3: 添加次要效果
1. 创建子对象添加额外粒子系统
2. 配置不同的参数创建层次感
3. 确保各系统协调工作

### 步骤4: 优化和测试
1. 调整性能参数
2. 在轨道系统中测试
3. 根据需要微调参数

### 步骤5: 保存为预制体
1. 拖拽到Project窗口保存
2. 为预制体添加合适的标签
3. 在文档中记录使用说明

## 🚨 常见错误和解决方案

### 问题1: 粒子不显示
**可能原因**:
- 发射率为0
- 粒子生命周期过短
- 材质设置错误
- Scale为0

**解决方案**:
- 检查Emission设置
- 增加Start Lifetime
- 验证材质和Shader
- 确认Transform设置

### 问题2: 性能问题
**可能原因**:
- 粒子数量过多
- 复杂的Shader
- 过多的模块

**解决方案**:
- 减少Max Particles
- 使用简单材质
- 禁用不必要的模块

### 问题3: 轨道同步问题
**可能原因**:
- Duration设置过短
- Simulation Speed不为1
- Play On Awake启用

**解决方案**:
- 增加Duration
- 设置Simulation Speed为1
- 禁用Play On Awake

---

💡 **提示**: 创建预制体时要始终考虑它在轨道系统中的使用场景，合理的预制体设计是获得优秀效果的关键。
